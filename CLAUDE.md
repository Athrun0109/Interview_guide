# AI 面试分析助手 - 项目设计文档

## 项目概述

开发一款"AI面试分析、攻略"应用，帮助求职者分析面试表现，提供改进建议和后续面试策略。

### 核心价值主张
- 完整的面试记录 >> 凭记忆的只言片语
- 为求职者提供客观的面试复盘分析
- 提供切实的改进建议和心理支持

---

## 开发阶段规划

### 第一阶段（当前 - Demo）
- **输入**: 面试文本内容（音频转文字）
- **目标**: 证明 pipeline 跑得通，AI 输出确实有用
- **用途**: 展示给老板获取开发资源

### 第二阶段
- **输入**: 纯音频（文本 + 语音语调分析）
- **目标**: 加入副语言特征分析

### 第三阶段
- **输入**: 视频内容（形象、动作、气质）
- **目标**: 全方位面试表现分析

---

## 技术架构

### Demo 技术栈
```
[音频文件] → [STT + 说话人分离] → [文本对话记录]
                                        ↓
[公司名 + 岗位 + JD] → [搜索摘要]  →  [组装 Prompt] → [LLM] → [分析报告]
```

### 组件选择

| 组件 | 选择 | 备注 |
|------|------|------|
| 前端 | Streamlit | 快速原型，Python 直出 Web UI |
| STT | faster-whisper | 本地 GPU 运行，支持英文/日文 |
| 说话人分离 | pyannote.audio | 2-3人场景效果可靠 |
| 搜索 | Serper.dev API | 免费额度 2500次/月 |
| LLM | Gemini Pro | 已有 API key |
| 运行环境 | 本地 GPU | Demo 阶段本地运行 |

### 语言支持优先级
1. 英文（主要）
2. 日文（次要）
3. 中文（暂时忽略）

## 核心功能设计

### 1. 说话人分离方案
- 使用声纹聚类（speaker embedding clustering）
- 输出每个 speaker 的样本片段
- **用户自选身份**：点选"这个是我"
- 其余 speaker 默认标记为"面试官"

### 2. 面试评价方式
- **定性评价**：good / bad / neutral（不使用数字评分）
- **逐条标注**：每个 Q&A pair 单独评价
- **附带解释**：每个评价有一两句话说明理由

### 3. 面试前信息收集
- Google 搜索公司背景（简化版）
- **用户输入**：
  - 公司名称（可选）
  - 岗位名称
  - 岗位 JD（招聘要求）

### 4. 优雅降级机制（重要）
当没有公司信息或可靠资料时：
- 退化为**纯面试内容分析**
- 仅基于面试对话给出评价和建议
- 不依赖外部搜索结果

### 5. 面试失败场景支持（重要）
当用户表示面试被拒时，输出内容调整为：
- **面试总结分析**
- **改进建议**（针对未来面试）
- **安慰和鼓励**（心理支持）

## Prompt 模板设计

### 标准分析模式
```
你是一位资深面试顾问。你的任务是分析一场面试对话，
帮助候选人了解自己的表现并改进。

## 背景信息
公司：{company_name}
公司简介：{search_summary}
岗位：{job_title}
岗位要求：
{job_description}

## 面试对话记录
{transcript}

## 分析要求
1. 将面试对话拆分为若干个问答环节
2. 对每个问答环节给出评价（好/一般/不好），并用一两句话说明理由
   - 评价标准：是否切题、是否展现了岗位要求的能力、表达是否清晰有条理
3. 推测面试官的关注点和提问逻辑
4. 给出 3 条最具体的改进建议，用于准备下一轮面试

用中文回答。语气专业但不刻板。
```

### 无公司信息模式
- 移除公司背景相关内容
- 聚焦面试技巧和表达分析

### 面试失败模式
- 调整语气为支持性
- 强调"这次经历的价值"
- 提供具体可行的改进方向
- 加入鼓励性语句

## 技术决策记录

### 为什么不需要"精炼模型"
- 60分钟面试 ≈ 12,000 tokens，远小于 Gemini 的 2M 上下文窗口
- 精炼模型会导致信息丢失和错误传播
- 通过 prompt 指令控制重点更有效

### 为什么选择定性评价而非定量评分
- 降低模型输出的验证门槛
- 更符合用户心智模型
- 减少 LLM 幻觉风险

### 第一阶段就需要的组件
即使第一阶段只用文本输入，**STT + diarization pipeline 也必须存在**，
因为需要将录音转写为带说话人标签的文本。
